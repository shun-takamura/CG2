# 🚀 ウェアラブル開発インフラシステム『J.A.R.V.I.S.』インプリメントプラン

~ Just A Remote Visual-Studio Infrastructure System ~

## 1. 開発コンセプト

本プロジェクトは、通学等の移動時間（歩行時・電車内）を100%開発時間に変えるため、手元のスマートフォン（ウェアラブル環境）から自宅の超ハイスペックリソース（RTX 4070 Ti SUPER / Visual Studio 2026）を安全に遠隔駆動し、自作DirectX 12ゲームエンジンの開発・デバッグを自律実行させる「次世代型開発インフラ」である。

- **ランタイムレス（隙間時間の完全活用）**: 自宅PCをサーバー化し、コンパイルやテストの待ち時間を移動時間に吸収。
- **タイプレス（キーボード不要）**: 音声入力（Typeless等）やDiscordのUIタップ（選択式）のみで、手癖と直感だけで高度なC++コーディングを行う。
- **リアルタイム目視デバッグ**: 画面配信（Go Live）や自動テスト実行を組み合わせ、歩きながら低レイヤの描画・同期バグを即座に確認・修正する。

## 2. システムアーキテクチャ（2面コンポーネント）

J.A.R.V.I.S.は、役割の異なる2つのAIパイプラインを統合して運用する。スマホ側のUIは「Discord（同じAPI・同一チャット画面）」に一元化する。

```
[移動中のスマホ (Discord / Typelessによる音声入力)]
                         │
                         ▼ (Tailscaleによる安全な通信)
            【自宅PC：J.A.R.V.I.S. (Python Bot)】
                         │
        ┌────────────────┴────────────────┐
        ▼ [指示の先頭に「[C]」や「クロード」]  ▼ [通常時 / リファクタリング時]
【タイプA：高度推論 / 直接編集】       【タイプB：物量作戦 / レビュー】
  - 脳みそ: Claude Code CLI             - 脳みそ: Ollama (Llama 3.1 / Qwen2.5)
  - コマンドラインから直接実行             - ローカルVRAM 16GBで無制限に駆動
  - プロファイルの直接自律書き換え          - 大規模レビュー、命名チェック等
        └────────────────┬────────────────┘
                         │
                         ▼
          【安全なGitブランチ運用＆検証】
           1. 自動的に main から新ブランチをチェックアウト
           2. ファイル自動書き換え実行
           3. MSBuild によるバックグラウンドコンパイル検証
           4. 【成功時】GitHub へ自動 Push
           5. 【成功時】テスト起動 ＆ スクショ撮影 ＆ Discord完了報告
```

### 🤖 タイプA：高度設計・直接ファイル編集（クラウド・Claude Code CLI連携型）

- **インフラ**: スマホ（Discord） ⇄ 自宅PC（Claude Code CLIツール：`claude` コマンド）
- **仕組み**: J.A.R.V.I.S.がプロンプトの先頭にある `[C]` や `クロード` のプレフィックスを検知すると、自宅PCのバックグラウンドで Claude CodeのCLI を直接子プロセスとして叩き、対象ファイルを直接編集させる。

```bash
claude "Renderer.cpp の〇〇をスレッドセーフになるよう書き換えて" --yes
```

※Proプランに紐づいたCLIを利用するため、追加のAPIキー従量課金は不要。

- **役割**: 複雑なDirectX 12のグラフィックスパイプライン設計、HLSLシェーダー同期、難解なバグのブレインストーミング。

### ⚙️ タイプB：物量作戦・自動リファクタリング（完全ローカル自律型）

- **インフラ**: スマホ（Discord） ⇄ 自宅PC（Ollama / Llama 3.1 8B ＆ Qwen2.5-Coder 14B）
- **役割**: API制限を一切気にせず無制限にコードをぶん回す。命名規則チェック（キャメルケース・スネークケース、メンバ変数の `m_` 付与）、大規模な誤字脱字チェック、冗長コードの削除など。

## 3. J.A.R.V.I.S. コアインターフェース（Discord スラッシュコマンド）

### ① `/prompt`（音声・自由記述プロンプトによるダイレクト開発）

- **引数**: `target_file`（対象ファイル名）, `instruction`（Typeless等による音声・テキスト指示）
- **挙動**: 指示のプレフィックスを自動判定。`[C]` があれば「Claude Code CLI」、それ以外は「ローカルLLM（Ollama）」のAPIを呼び出してファイルを書き換える。書き換え後は自動的に【安全なGit運用フロー】に移行する。

### ② `/refactor`（定型クリーンアップ）

- **引数**: `target_file`
- **挙動**: あらかじめシステムプロンプトに仕込んだ「コーディング規約」に基づき、ローカルLLMをフル活用してコードレビュー、変数名修正、冗長コードの削除を一発で実行する。

### ③ `/test_run`（低レイヤ自動入力テスト＆リアルタイム配信）

- **引数**: `keys`（シミュレートキー、コンマ区切り）、`wait_sec`（起動から撮影までの待機秒数）
- **挙動**:
  - `pydirectinput` を使い、DirectX 12ウィンドウに低レイヤのキー信号（スキャンコード）をシミュレート（例：スペースキーでのジャンプ）。
  - `pyautogui` で描画結果のスクリーンショットを撮影し、Discordに画像を添付して返信する。
- **発展機能（Go Liveハック）**: 自宅PCのDiscordをAPIや仮想カメラ経由でフックし、ゲーム起動時に自動でボイスチャンネルへ画面配信（Go Live）を開始する。スマホでボイチャに接続するだけで、リアルタイムでゲームエンジンの動作（60fps）を目視確認できる。

## 4. 🔒 鉄壁のGitブランチ隔離・自動ビルド運用フロー

AIによる main ブランチの破壊を防ぎ、安全性を担保する自動化パイプライン。

1. **ブランチ自動生成**: コマンド受信時、最新の main から `ai-refactor-YYYYMMDD-HHMM` ブランチを自動で切り替える。
2. **AI書き換え実行**: 各タイプ（A/B）が対象のソースコードを書き換える。
3. **MSBuild自動コンパイル**: Visual Studio 2026の `msbuild` コマンドをバックグラウンドで実行。

```bash
msbuild YourEngine.sln /p:Configuration=Debug /p:Platform=x64
```

4. **安全弁（ビルドチェック）**:
   - **ビルド成功時**: その一時ブランチにコミットを作成し、GitHubのリモートへ自動プッシュ。
   - **ビルド失敗時**: 処理を安全に中断。コンパイルエラーログをテキストとしてDiscordに送信し、スマホ側に通知する。
5. **人間の最終レビュー**: 学校に到着後、ノートPCでGitHubを開き、AIが作成したプルリクエスト（PR）の差分（Diff）を目視確認。問題がなければ手動で main へマージする。

## 🗺️ 開発・実装ロードマップ（合格基準付き）

### 【Step 1：ローカルPCのインフラ検証（すべての土台）】

**目的**: AIやDiscordを繋ぐ前に、自宅PC単体で「自動化コマンド」が通る環境を作る。

- [ ] VS2026のコマンドプロンプトで `msbuild YourProject.sln` を実行し、画面を開かずにビルドが通るか検証。
- [ ] 簡易Pythonスクリプトからゲームの `.exe` を起動し、`pydirectinput` でキー入力を送り、`pyautogui` でスクショが撮影されて正常終了することを確認。
- [ ] `Tailscale` をスマホと自宅PCに導入し、外（4G/5G）から自宅PCのローカルIPに接続できることを確認。

★ **合格基準**: Pythonを1回実行すると、ゲームが自動起動・自動入力テストされ、スクショが保存されて正常終了する。

### 【Step 2：安全なGitパイプラインの自動化】

**目的**: 手動で行っていたGit操作をPythonに身代わりさせる。

- [ ] `git checkout main` ➜ `git pull` ➜ 新ブランチ作成 ➜ 模擬修正 ➜ `msbuild` 実行 ➜ 成功時にコミット＆GitHubへ自動プッシュ、という一連の流れを行うPythonスクリプトを作成。

★ **合格基準**: 手動でPythonを実行すると、GitHub上にビルド確認済みの新しいブランチが自動生成されている。

### 【Step 3：J.A.R.V.I.S. の脳みそ（LLM & Claude CLI）の接続】

**目的**: 各種AIをスクリプトから呼び出し、ファイルを直接編集する。

- [ ] 自宅PCに `Ollama`（Qwen2.5-Coder / Llama 3.1）を導入し、PythonからAPIで呼び出してファイルを置換できる仕組みを作る。
- [ ] コマンドプロンプトから `claude "指示" --yes` を実行した際、プロジェクトファイルが自動で直接編集されることを確認。
- [ ] 指示文のプレフィックス（`[C]` など）に応じて、OllamaとClaude CLIの実行を分岐するコードを実装。

★ **合格基準**: 「〇〇を直して」とスクリプトに渡すと、AIが自動でファイルを書き換え、コンパイルを通し、Gitにプッシュする。

### 【Step 4：Discordインターフェースの実装（スマホ連動）】

**目的**: スマホからDiscord経由でJ.A.R.V.I.S.を自在に操る。

- [ ] `discord.py` を用いて、自宅PC側で常時待機するBotを起動。
- [ ] `/prompt`, `/refactor`, `/test_run` のスラッシュコマンド（引数付き）を実装。
- [ ] 処理完了時、ビルド成否ステータス、Gitのブランチ名、撮影したスクショをDiscordに @メンション 付きで返信する定型メッセージを実装。

★ **合格基準**: スマホからDiscordで指示を出し、数分後にビルド成否とスクショ画像がピコンとスマホに届く。
